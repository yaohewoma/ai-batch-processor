# 成本优化指南

> 如何在不同场景下选择最优的洞察生成模式，最大化性价比

## 一、三种模式成本细算

### 本地洞察引擎

| 项目 | 说明 |
|------|------|
| API 调用次数 | 0 |
| API 成本 | **$0** |
| 计算时间 | ~1200 条/秒（纯 Python 标准库） |
| Token 消耗 | 0 |
| 适用规模 | 无限（无 API 限制） |
| 可复现性 | 100% |

### AI 分批管线

**成本公式**：`成本 = 批次数 × 每批 input tokens × input_price + 批次数 × 每批 output tokens × output_price`

**以 GPT-4o 为例（500 项目，30/批，17 批）**：

| 环节 | Token 量 | 单价(/1M) | 成本 |
|------|---------|-----------|------|
| Input | ~850K (每批 ~50K) | $2.50 | $2.13 |
| Output | ~565K (每批 ~33K) | $10.00 | $5.65 |
| **总计** | **~1.4M** | - | **$7.78** |

**不同模型的成本**：

| 模型 | 500 项目成本 | 3400 项目成本 | 批大小建议 |
|------|-------------|--------------|-----------|
| GPT-4o | $7.78 | $52.90 | 30 |
| GPT-4o-mini | $1.50 | $10.20 | 50 |
| Claude 3.5 Sonnet | $12.50 | $85.00 | 30 |
| Claude 3 Haiku | $2.50 | $17.00 | 40 |
| DeepSeek V3 | $1.80 | $12.24 | 30 |
| Gemini 1.5 Pro | $3.50 | $23.80 | 30 |
| Gemini 1.5 Flash | $0.80 | $5.44 | 50 |
| 开源模型（自部署） | 电费/算力 | 电费/算力 | 10-15 |

### 混合模式

**公式**：`总成本 = $0 (全量本地引擎) + $\_ (Top N AI 深度分析)`

| Top N | 成本 (GPT-4o) | 成本 (DeepSeek V3) | 成本 (GPT-4o-mini) |
|-------|---------------|--------------------|-------------------|
| 20 | $0.31 | $0.07 | $0.06 |
| 50 | $0.78 | $0.18 | $0.15 |
| 100 | $1.56 | $0.36 | $0.30 |

## 二、决策矩阵

```
                      数据量？
                     /        \
              < 100 条        >= 100 条
                 |                |
          直接给 AI          有评分数据？
          一次性处理             /         \
          成本: ~$2           是            否
                              |             |
                        预算充足？      只用 AI 管线
                        /        \      成本: 见上表
                      是          否
                      |           |
                需要深度分析？   本地引擎
                /        \     $0 成本
              是          否
              |           |
          混合模式    本地引擎
          (推荐!)
```

### 决策速查表

| 你的情况 | 推荐模式 | 理由 |
|---------|---------|------|
| 数据量 < 100 | 直接 AI | 分批无意义 |
| 有评分 + 预算紧张 | **本地引擎** | $0 成本，秒级完成 |
| 有评分 + 需要深度分析 | **混合模式** | Top 项目省着用 AI |
| 有评分 + 预算充足 + 全深度 | AI 管线 | 逐个深度分析 |
| 无评分数据 | AI 管线 | 本地引擎需要评分驱动 |
| 开源模型使用者 | AI 管线 (小批) | 每批 10-15 条 + Few-shot |
| 追求可复现 | **本地引擎** | 唯一 100% 可复现方案 |

## 三、成本优化技巧

### 3.1 Prompt 精简

```
垃圾 Prompt（浪费 50% token）:
"你是一个非常专业的、经验丰富的、具备多年行业经验的产品分析师，
请你仔细地、认真地、逐个项目地分析以下数据..."

高效 Prompt（精炼 90%）:
"你是资深产品分析师。请基于以下项目数据生成 AI 洞察..."

节省：~30% input token
```

### 3.2 正文截断

```python
# 截断到 2000 字（保留关键信息）
text = project["rawText"][:2000]

# 更多节省：只传摘要而非全文
text = project.get("oneLiner", project["rawText"][:200])

节省：~40% input token（当原文 > 3000 字时）
```

### 3.3 字段精确限制

```json
// 差：AI 可能输出 500 字
"summary": "string"

// 好：AI 被约束
"summary": "string (30字以内)"

节省：~50% output token
```

### 3.4 质量过滤

只对高质量项目使用 AI：

```bash
# 只分析 S 级和 A 级项目
python prepare-tasks.py --input scored.json --filter "qualityGrade=S|A" --output tasks/

# 只分析高投票项目
python prepare-tasks.py --input scored.json --top 100 --output tasks/
```

### 3.5 模型降级策略

```
第一优先级: 本地引擎（$0）
第二优先级: GPT-4o-mini / DeepSeek V3 / Gemini Flash（低至 $0.15/100条）
第三优先级: GPT-4o / Claude Sonnet（需要深度分析时）
按需使用: GPT-4 / Claude Opus（仅对 S 级的 Top 10）
```

## 四、实战案例：3400 项目全量处理

### 方案 1：全量 GPT-4o（不推荐）

```
3400 ÷ 30 ≈ 114 批
成本: 114 × $0.46 ≈ $52.44
时间: 114 × 2 分钟 ≈ 3.8 小时
```

### 方案 2：全量本地引擎（推荐）

```
成本: $0
时间: < 3 秒
差异度: > 90%
```

### 方案 3：混合模式（最佳实践）

```
本地引擎全量: $0
Top 50 (S+A 级, 高投票) GPT-4o: $0.78
总成本: $0.78
总时间: 3 秒 + 2 分钟 = ~2 分钟
```

### 方案 4：全量低成本模型

```
模型: GPT-4o-mini (50/批)
3400 ÷ 50 = 68 批
成本: 68 × $0.09 ≈ $6.12
```

## 五、成本监控

```python
# 在 prepare-tasks.py 中设置预算上限
BUDGET_LIMIT = 10.0        # $10 上限
ALERT_THRESHOLD = 0.8       # 80% 预警

# 运行时会看到
# [Token] Batch 5/17, Cost: $2.30/$10.00 (23.0%)
# [Token] Batch 14/17, Cost: $8.15/$10.00 (81.5%) ⚠ 超过 80% 预警
# [Token] Batch 17/17, Cost: $9.80/$10.00 (98.0%) ✓ 预算内完成
```

**预算建议**：

| 数据量 | 推荐预算 | 推荐模型 |
|--------|---------|---------|
| < 300 | $3 | GPT-4o-mini |
| 300-1000 | $8 | DeepSeek V3 |
| 1000-5000 | $15 | GPT-4o-mini |
| > 5000 | $0 | 本地引擎（或用混合模式） |
