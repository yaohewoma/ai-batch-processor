# Prompt 模板设计

## 模板结构

有效的 Prompt 应包含以下要素：

1. **角色设定**：明确 AI 的身份和分析视角
2. **输出 Schema**：严格 JSON Schema，包含字段名、类型、字数限制
3. **分析维度**：引导 AI 思考方向
4. **数量提示**：告知这批有多少条数据，方便输出校验
5. **Few-shot 示例**（推荐）：提供 1-2 个示例帮助 AI 理解格式和风格

## 完整模板

```python
PROMPT_TEMPLATE = """你是一个资深的产品分析师。请基于以下项目数据，为每个项目生成AI洞察报告。

要求：
1. 为每个项目输出一个JSON对象，包含以下字段：
   - topicId: 项目ID（字符串）
   - aiInsight: 包含以下结构的洞察对象：
     - summary: 一句话总结（30字以内）
     - competitiveEdge: 核心竞争优势分析（50-80字）
     - risks: 潜在风险点（列表，2-3个，每个30-50字）
     - suggestions: 改进建议（列表，2-3个，每个30-50字）
     - marketOpportunity: 市场机会分析（50-80字）

2. 分析维度：
   - 功能性：功能是否完整、是否解决真实问题
   - 创新性：是否有独特卖点、是否与竞品差异化
   - 用户体验：交互设计是否友好
   - 技术难度：技术实现复杂度
   - 实用性：实际应用场景、用户价值

3. 输出格式：
   请输出一个JSON数组，每个元素是一个项目的洞察对象。
   确保JSON格式正确，可以直接被解析。

以下是第{batch_id}批的项目数据（共{count}个）：

{projects_json}

请开始分析并输出JSON数组："""
```

## Few-shot 示例

在 Prompt 末尾添加 1-2 个示例，可显著提升 AI 输出质量：

```python
FEWSHOT_EXAMPLE = """
示例输出格式（请严格遵循此格式）：
[
  {
    "topicId": "123",
    "aiInsight": {
      "summary": "面向开发者的AI编程助手，功能完整度突出",
      "competitiveEdge": "功能完整度（8.2分）表现优秀，获120票社区支持验证了产品价值，技术栈涵盖Python、LLM应用等，架构有深度",
      "risks": [
        "用户体验（5.5分）有待优化，新用户上手可能有门槛",
        "仅15票，用户基数较小，规模化增长面临挑战"
      ],
      "suggestions": [
        "优化用户体验（当前5.5分），降低新用户上手成本",
        "积极在社区互动推广，提升作品曝光度"
      ],
      "marketOpportunity": "AI编程开发赛道持续发展，AI赋能的产品有差异化突围机会，早期用户反馈可指导产品快速迭代和方向调整"
    }
  }
]
"""
```

## 模型适配说明

### 不同模型的 Prompt 策略差异

| 模型 | 推荐策略 | 注意事项 |
|------|----------|----------|
| **GPT-4o / GPT-4** | 标准 Prompt 即可，可适当放宽每批条数到 50 | JSON 输出稳定性高，偶尔需要处理 markdown 包裹 |
| **Claude 3.5 Sonnet** | 标准 Prompt，建议添加 `{` 开头提示 | 对 JSON Schema 约束理解较好，但需要明确类型 |
| **Claude 3 Haiku** | 减小每批条数到 15-20，加强字段约束 | 小模型容易遗漏字段，建议添加 Few-shot 示例 |
| **DeepSeek V3** | 标准 Prompt，建议减少字数限制的矛盾表述 | 中文理解优秀，但 JSON 严格性略低于 GPT-4 |
| **Gemini 1.5 Pro** | 标准 Prompt，建议增加 Few-shot 示例 | JSON 输出格式较好，但偶尔省略必要字段 |
| **本地/开源模型 (Qwen, Llama 等)** | 必须添加 Few-shot 示例，减小每批到 10-15 条 | JSON 格式易出错，建议增强后处理容错 |

### 调优建议

- **加强 JSON 约束**：在 Prompt 末尾添加 `请确保输出是严格的 JSON 数组，不要包含任何额外文字。`
- **温度参数**：建议 temperature 设为 0.3-0.5，保证输出一致性
- **重试机制**：对 JSON 解析失败的批次，可让 AI 重新生成（附上格式要求）

## 字数限制技巧

```
- summary: 一句话总结（30字以内）          ← 精炼，AI 不会啰嗦
- competitiveEdge: 核心竞争优势（50-80字）  ← 范围，AI 不会太短或太长
- risks: 列表，2-3个                       ← 数量，AI 不会只输出1个
```

## 给 AI 传参考数据

```python
def simplify_for_ai(project):
    """传已有的评分 + 优劣势，AI 可参考"""
    return {
        "topicId": project["topicId"],
        "title": project["title"],
        "avgScore": project["avgScore"],       # 已有评分
        "strengths": project["strengths"][:3],  # 已有优势
        "weaknesses": project["weaknesses"][:3], # 已有劣势
        "rawText": project["rawText"][:2000],   # 截断正文
    }
```

## 常见错误

| 错误 | 后果 | 修复 |
|------|------|------|
| 不限制字数 | AI 输出 500 字，token 浪费 | 每个字段标注字数范围 |
| Schema 不明确 | AI 输出中文 key 而非英文 | 明确指定字段名和类型 |
| 不限制数量 | AI 输出 1 个建议或 10 个建议 | 标注"2-3个" |
| 不传参考数据 | AI 凭空分析，与已有评分矛盾 | 传已有评分和标签 |
| 角色设定太泛 | AI 输出偏通用 | 明确"资深产品分析师"等具体角色 |
| 缺少 Few-shot 示例 | 小模型/开源模型输出格式不统一 | 添加 1-2 个 JSON 输出示例 |
